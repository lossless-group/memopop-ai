# Changelog - 2025-12-27 (#1)

## MemoPop AI: Monorepo Foundation

### Overview

Created the `lossless-group/memopop-ai` public repository and established a production-grade monorepo structure using Bun workspaces. This provides the foundation for three apps working together: a marketing site, a web application, and an AI agent orchestrator.

**Key deliverables:**

- **Monorepo scaffold** with Bun workspaces (no Turborepo needed yet)
- **Astro marketing site** with full landing page ported from GitHub Pages
- **SvelteKit web app** skeleton with Svelte 5 runes syntax
- **Shared Tailwind config** as a workspace package

---

## Architecture Decisions

These decisions were documented in `context-v/Vision-for-Production-Grade-Memopop-Monorepo.md`:

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Package manager | Bun + Bun workspaces | Fast, modern, native workspace support. Turborepo as escape hatch if builds slow down. |
| Frontend (marketing) | Astro | Best static site perf, content collections, no React |
| Frontend (app) | SvelteKit + Svelte 5 | Modern reactivity with runes, no React patterns |
| Styling | Shared Tailwind | Single config in `packages/shared-styles/`, both apps consume |
| Deployment | Vercel | Existing paid account, great Astro + SvelteKit support |
| Auth | SuperTokens on Railway | Open source, self-hosted, full control |
| Database | Baserow on Railway | Open source, self-hosted, easy data management |
| Changelog | Distributed per app | Each app has own changelog, aggregated via GitHub raw API |

## Landing Page Port

Ported the full MemoPop landing page from the GitHub Pages `docs/` folder to Astro:

### Sections Implemented

1. **Navigation** - Sticky header with mobile hamburger menu
2. **Hero** - Headline, subheadline, CTAs, animated memo preview
3. **Problem** - Pain point validation with bullet list
4. **How It Works** - 3-step process cards
5. **Features** - 7 feature cards including "Voice of the Firm" highlight
6. **Output Showcase** - Dark mode memo mockup with citation callout
7. **Agent Architecture** - Flow diagram of 5 key agents
8. **Pricing** - 3-tier pricing cards (Starter/Pro/Enterprise)
9. **FAQ** - 6-item accordion using native `<details>`
10. **Final CTA** - Gradient background with action button
11. **Footer** - Brand, links, social icons

### Styling Approach

- **CSS Variables** in `BaseLayout.astro` for theming
- **Scoped styles** in `index.astro` for page-specific CSS
- **Mobile-first** responsive design
- **No Tailwind utility classes yet** - CSS ported directly for speed

## Shared Styles Package

Created `@memopop/shared-styles` as a workspace package:

```js
// packages/shared-styles/tailwind.config.js
export default {
  theme: {
    extend: {
      colors: {
        primary: '#1a3a52',
        secondary: '#1dd3d3',
        accent: '#f59e0b',
        // ... full palette
      },
      fontFamily: {
        sans: ['Inter', 'system-ui', 'sans-serif'],
        mono: ['JetBrains Mono', 'monospace'],
      },
    },
  },
};
```

Both apps reference via workspace protocol:
```json
{
  "dependencies": {
    "@memopop/shared-styles": "workspace:*"
  }
}
```

## SvelteKit App Skeleton

Initialized with Svelte 5 runes syntax:

```svelte
<script lang="ts">
  // Svelte 5 runes
  let count = $state(0);

  function increment() {
    count++;
  }
</script>

<button onclick={increment}>
  Count: {count}
</button>
```

Configured with:
- Vercel adapter for deployment
- TypeScript strict mode
- Workspace dependency on `@memopop/shared-styles`

```bash
# Build both
bun run build
```
